Qu’est-ce que la réactivité ? La capacité de Vue.js à détecter les changements dans les données et à mettre à jour automatiquement le DOM. Comment Vue.js gère-t-il la réactivité ? À l'aide d'un système basé sur des observateurs (observers). Utilise une structure appelée Proxy pour "écouter" les modifications. Stockage d’une valeur réactive = ref() permet de crée une valeur réactive qui peut être un type primitif (comme un nombre, une chaîne, etc.) ou un objet. Les modifications de cette valeur déclenchent une mise à jour du DOM si elle est utilisée dans un composant. • La valeur est accessible via .value • Adapté pour des valeurs primitives et des objets simples. Objet réactif reactive() transforme un objet en un objet réactif profond. Toutes les propriétés de cet objet deviennent réactives. • Utilisé pour des structures plus complexes (objets ou tableaux). • Les modifications des propriétés de l'objet déclenchent des mises à jour. Calcul de valeurs dérivées computed() crée une valeur réactive dérivée d'autres valeurs réactives. Elle est mise en cache et ne se met à jour que lorsque ses dépendances changent. • Idéal pour éviter de recalculer des valeurs inutiles. • Peut être utilisée en lecture seule ou comme propriété "getter-setter". L’état représente les données partagées et manipulées dans une application. Problème sans gestion d’état : • Difficulté à synchroniser les données entre plusieurs composants. • Complexité accrue dans les applications complexes. Pourquoi gérer l’état ? : • Faciliter la communication entre composants. • Garantir la cohérence des données dans toute l'application. Approches natives de Vue.js Communication via Props et Events : • Props : Passer des données des composants parents aux enfants. • Events : Permettre aux enfants d'envoyer des informations aux parents. •Limites des Props et Events : • Devient difficile à gérer lorsque l’application contient de nombreux composants. • Nécessité d’un outil centralisé pour gérer l’état. Vuex ou Pinia Vuex : • Une bibliothèque officielle pour la gestion d’état dans Vue.js. • Basée sur un store centralisé qui gère les données partagées. • Fonctionne selon des concepts tels que state, getters, mutations, et actions. Pinia : • Une alternative moderne à Vuex, plus légère et intuitive. • Meilleure intégration avec la Composition API. • Moins de boilerplate (code répétitif). Simplification de Pinia Pourquoi Pinia est plus simple ? • Pas besoin de this.$store : Avec Pinia, chaque store est une fonction que l'on importe directement dans le composant. • Pas de commit ni de dispatch : Les actions et mutations sont unifiées dans des méthodes simples. • Approche modulaire : Chaque store est indépendant et organisé autour d’une logique métier spécifique. Pourquoi utiliser Pinia Gestion centralisée de l’état : • Toutes les données partagées sont regroupées dans des stores, facilitant leur accès et leur manipulation. Facilité de maintenance et débogage : • Une API simple et intuitive permet une lecture et une compréhension rapide du code. • Compatible avec les outils modernes de débogage (Vue DevTools). Fonctionnalités avancées prises en charge : • Time-travel debugging : Naviguez dans les différentes étapes de l’état pour analyser les erreurs. • Compatibilité avec Vue 3 : S’intègre parfaitement avec la Composition API. Modularité : • Possibilité de créer plusieurs stores indépendants, simplifiant la gestion des états dans des applications complexes. Qu’est-ce que Material Design ?
